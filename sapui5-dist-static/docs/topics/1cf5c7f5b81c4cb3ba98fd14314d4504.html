List Report Elements

The list report lets the user work with a large list of items. It combines powerful functions for filtering large lists with different ways of displaying the resulting item list.

This image is interactive. Hover over each area for a description. Click highlighted areas for more information

Main Elements of the List Report

The list report view includes the following main elements:

  • Application header

  • Filter bar with variant management and a generic Share menu that includes the following actions:

    • Send Email

    • Save as Tile

  • Table

  • Footer toolbar that can include optional actions

Loading Data

When you launch the application, the system loads the data automatically to the list report by default and the smart filter bar appears collapsed if one or more filter fields have some default values.

Note

Provide default values for filter fields to avoid showing empty tables when you launch the app.

Autoloading is the default behavior for the standard variant if any default filter values have been defined. Application developers can change this behavior through manifest settings.

SAP Fiori Elements for OData V2

The following settings can be made:

  • Application Developers:

    Application developers can change the default behavior using the loadDataOnAppLaunch property in the manifest setting as shown below:

    Sample Code 

    “pages”: {
               “ListReport | < EntitySet > “: {
               		“entitySet”: < EntitySet > ,
                   	 “component”: {
                                  	 “name”: “sap.suite.ui.generic.template.ListReport”,
                                  	 "list": true,
                                  	 “settings”: {
                                                 "dataLoadSettings": {
                                                      "loadDataOnAppLaunch": "always" / "never"/ "ifAnyFilterExist"
                                                   }
                                         }
                           }
                  }

    loadDataOnAppLaunch can be set to always, never, ifAnyFilterExist or left undefined.

    The initial app data is loaded based on the following settings:

    • always: Data for the standard variant is always loaded. End users can override this by deselecting the Apply Automatically checkbox in the Manage Views dialog.

    • never: Data for the standard variant is not loaded automatically, and the user has to actively click the Go button. End users can override this by selecting the Apply Automatically checkbox in the Manage Views dialog.

    • ifAnyFilterExist is set or is not defined: Data for the standard variant is loaded only if the default filter values have been defined (by an administrator or a user, or passed from another app). This is the default behavior for any newly generated apps. End user cannot override this default behavior in Manage Views dialog. The Apply Automatically checkbox is not available, rather the text For Preset Filters is displayed.

    These settings work for both single and multi-view applications.

    For multi-view scenarios, you can also change the manifest setting enableAutobinding to False under quickVariantSelectionX instead of using the loadDataOnAppLaunch setting. However, we recommend you to use the manifest setting loadDataOnAppLaunch for the multi-view scenario as well.

    Sample Code 

    “pages”: {
          “ListReport|<EntitySet> “: {
                  “entitySet”: <EntitySet>,
                  “component”: {
                  “name”: “sap.suite.ui.generic.template.ListReport”,
                  “settings”: {
                              “quickVariantSelectionX”: {
                                      “enableAutoBinding”:  false
                               }
                   }
             }
          }

  • End Users:

    End users can disable autoloading by deselecting the Apply Automatically checkbox in the Manage Views dialog. To enable autoloading, select the Apply Automatically checkbox. This is not possible if loadDataOnAppLaunch is not defined or is set to ifAnyFilterExist.

Please note that the smart filter bar with variant management and a generic Share menu also includes the action Share in SAP Jam (if the integration with SAP Jam is configured).

SAP Fiori Elements for OData V4

The following settings can be made:

  • Application Developers:

    Application developers can change the default behavior using the initialLoad property in the manifest setting as shown below:

    Sample Code 

    "sap.ui5": {
	...
	"routing": {
		"routes": [...],
		"targets": {
			"SalesOrderManageList": {
				"type": "Component",
				"id": "SalesOrderManageList",
				"name": "sap.fe.templates.ListReport",
				"options": {
					"settings": {
						"entitySet": "SalesOrderManage",
						"variantManagement": "Page",
						"initialLoad": "Auto",
						"controlConfiguration": {...},
						"navigation": {...}
					}
				}
			},
			...
		}
	},
	...
}

    initialLoad can be set to Auto, Disabled, and Enabled.

    The app data is loaded based on the following settings:

    • Auto: An initial load of data only occurs if some default filter values have been set in the filter bar. The Apply Automatically checkbox is replaced by the text For Preset Filters in the Manage Views dialog. This is the default setting.

    • Disabled: An initial load of data does not occur for the standard variant and the user has to actively click the Go button. End users can override this by selecting the Apply Automatically checkbox in the Manage Views dialog.

    • Enabled: An initial load of data occurs for the standard variant. End users can override this by deselecting the Apply Automatically checkbox in the Manage Views dialog.

    These settings work for both single and multi-view applications.

  • End Users:

    End users can disable autoloading by deselecting the checkbox Apply Automatically in the Manage Views dialog. To enable autoloading, end users can select the checkbox Apply Automatically in the Manage Views dialog.

    If initialLoad is set to Auto, then the checkbox for Apply Automatically is replaced by the text For Preset Filters in the Manage Views dialog.
