Multi-Selection for Intent-Based Navigation

By default, multi-selection for intent-based navigation is enabled.
Note

The records which do not have a unique key field cannot be identified. For example, if the following contexts are to be passed during navigation:

Record

P1

P2

1

A

X

2

B

Y

3

A

Y
The navigation context has P1 = A,B and P2 = X,Y. In the target app, individual properties are concatenated via AND and multiple property values are concatenated via OR: (P1=A OR P1=B) AND (P2=X OR P2=Y). This might result in an additional record BX.

Note

Please note that this topic is currently only applicable to SAP Fiori elements for OData V4.
