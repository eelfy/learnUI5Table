Using the Global Side Effect

To optimize performance, make the annotations for the desired side effects as specific as possible.

You can define a side effect without any source properties or source entities. This is called the global side effect. In this case, when you choose the Enter key, the side effect is triggered. If no target is defined, it triggers the refresh of the whole entity. This is not recommended. You can avoid this by specifying the target properties or target entities, and forceGlobalRefresh parameter. The forceGlobalRefresh parameter in the manifest.json defines the global refresh behavior.

Sample Code 

"sap.ui.generic.app": {
        "_version": "1.3.0",
        "settings": {
            "forceGlobalRefresh": false,
            "inboundParameters": {
                "Supplier": {
                    "useForCreate": true
                }
            },

  • If the parameter is set to true, a global refresh is triggered when the user chooses Enter, even if a side effect with no source properties or target have been annotated..

  • If the parameter is set to false, the system reacts according to what has been annotated.

  • If the parameter is not set, for compatibility reasons, the (runtime) default is true.

For newly generated apps, the (design time) default is false, that is, the project generation wizard generates "forceGlobalRefresh": false.

Sample Code

Example of a global side effect

<Annotations Target="NAMESPACE.ENTITYTYPE">
<Annotation Term="com.sap.vocabularies.Common.v1.SideEffects" Qualifier="GlobalSideEffect">
   <Record>
      
      <PropertyValue Property="TargetProperties">
         <Collection>
            <String>TaxAmount</String>
            <String>NetAmount</String>
            <String>GrossAmount</String>
         </Collection>
      </PropertyValue>
   </Record>
</Annotation>
</Annotations>

Note

Global side effect is currently only applicable for draft applications.
