Setting Default Filter Values

Setting Default Filter Values

You can set default filter values in list report applications, for example.

SAP Fiori Elements for OData V2

You can set default filter values in list report and analytical list report applications.

Using the SelectionVariant Annotation

You can set default values for smart filter bar fields using SelectionVariant annotation in the annotationPath app descriptor settings.

The default values from the SelectionVariant are displayed on application load for standard variant. You can set more than one value as a default value for a filter field.

Sample Code

Example with a qualified path

"ListReport|C_STTA_SalesOrder_WD_20": {
      "entitySet": "C_STTA_SalesOrder_WD_20",
      "component": {
            "name": "sap.suite.ui.generic.template.ListReport",
            "list": true,
            "settings": {
            "annotationPath": "com.sap.vocabularies.UI.v1.SelectionVariant#FilterDefaults"
            }
      }
}

<Annotation Term="UI.SelectionVariant" Qualifier="FilterDefaults">
   <Record>
      <PropertyValue Property="SelectOptions">
         <Collection>
            <Record Type="UI.SelectOptionType">
               <PropertyValue Property="PropertyName" PropertyPath="so_id" />
               <PropertyValue Property="Ranges">
                  <Collection>
                     <Record Type="UI.SelectionRangeType">
                        <PropertyValue EnumMember="UI.SelectionRangeSignType/I" Property="Sign" />
                        <PropertyValue EnumMember="UI.SelectionRangeOptionType/EQ" Property="Option" />
                        <PropertyValue Property="Low" String="500009447" />
                     </Record>
                     <Record Type="UI.SelectionRangeType">
                        <PropertyValue EnumMember="UI.SelectionRangeSignType/I" Property="Sign" />
                        <PropertyValue EnumMember="UI.SelectionRangeOptionType/EQ" Property="Option" />
                        <PropertyValue Property="Low" String="500000841" />
                     </Record>
                  </Collection>
               </PropertyValue>
            </Record>
         </Collection>
      </PropertyValue>
   </Record>
</Annotation>

Note

  • This configuration overrides all the default values coming from the Common.FilterDefaultValue annotations.

  • The manifest property annotationPath takes both qualified and unqualified annotation paths.

  • The default values through the annotationPath property can only be set via the UI.SelectionVariant annotation.

  • .If the UI.SelectionVariant annotation configured in the annotationPath does not exist, an error page is loaded.

Using the Common.FilterDefaultValue Annotation

You can configure default filter values by defining the Common.FilterDefaultValue annotation for a property type. You can set only one value as a default value for a filter field.

Note

The default filter values from the Common.FilterDefaultValue property are overwritten by the variant management or navigation context in the specified order.

<Annotations Target="STTA_PROD_MAN.STTA_C_MP_ProductType/Supplier">
    <Annotation Term="Common.FilterDefaultValue" String="100000047"/>
</Annotations>
Note

If you configure default filters via UI.SelectionVariant annotation using the manifest property annotationPath, all other filters configured via Common.FilterDefaultValue annotation are ignored.

If you define user default filter values in SAP Fiori Launchpad, they are combined with other default values. The priority is as following:

Parameters coming from... Result
Navigation context Overrides user defined default variant and standard variant. The navigation context is applied.
User defined default variant (This variant is not equal to the standard variant and there is no navigation context.) Overrides everything, including the SAP Fiori launchpad user default parameters.
Only standard variant (There is no navigation context and no other variant defined.) combined with optional UI.SelectionVariant or Common.FilterDefaultValue annotations User defined default parameters are applied and combined with the values from the standard variant.

SAP Fiori Elements for OData V4

For the standard variant, application developer can configure default filter values by using the SelectionVariant annotation or the Common.FilterDefaultValue annotation.

Using the SelectionVariant Annotation

You can either provide the UI.SelectionVariant annotation directly, or as part of the UI.SelectionPresentationVariant. For more information, refer to Creating a List Report Without Variant Management.

If you provide the UI.SelectionVariant, then the default filter values are applied for the filter fields in accordance with what is specified via the UI.SelectionVariant annotation. In the sample code below, the filter field OverallSDProcessStatus is applied with value 'A':

Sample Code

Default Filter Values via UI.SelectionVariant Annotation

<PropertyValue Property="SelectionVariant">
    <Record Type="UI.SelectionVariantType">
        <PropertyValue Property="Text" String="Open"></PropertyValue>
        <PropertyValue Property="SelectOptions">
            <Collection>
                <Record Type="UI.SelectOptionType">
                    <PropertyValue Property="PropertyName" PropertyPath="OverallSDProcessStatus" />
                    <PropertyValue Property="Ranges">
                        <Collection>
                            <Record Type="UI.SelectionRangeType">
                                <PropertyValue Property="Option" EnumMember="UI.SelectionRangeOptionType/EQ" />
                                <PropertyValue Property="Low" String="A" />
                            </Record>
                        </Collection>
                    </PropertyValue>
                </Record>
            </Collection>
        </PropertyValue>
    </Record>
</PropertyValue>

Sample Code

CDS Annotation

SelectionVariant : {
 Text: 'Open',
 SelectOptions: [
    { $Type : 'UI.SelectOptionType',
      PropertyName : OverallSDProcessStatus,
      Ranges: [
      {
        $Type: 'UI.SelectionRangeType',
        Option: #EQ,
        Low: 'A'
      }
    ]
   }
 ]
}

Using the Common.FilterDefaultValue Annotation

If only single values need to be applied for the filter fields, application developers can use the Common.FilterDefaultValue annotation.

<Annotations Target="STTA_PROD_MAN.STTA_C_MP_ProductType/OverallSDProcessStatus ">
    <Annotation Term="Common.FilterDefaultValue" String="A"/>
</Annotations>

Sample Code

CDS Annotation

OverallSDProcessStatus        : String(1)                           @(
            Core.Immutable : true,
            Common         : {
                FilterDefaultValue      : 'A'
            }
);

Note

This does not allow complex values ('Supplier StartsWith "AB"') or multiple values ('Status = "A" or Status = "B"'). Please note the following behavior:

  • If the SelectionVariant is provided, then this is used and the Common.FilterDefaultValue is ignored for all other filter fields.

  • The defaults via these annotations are only applied for the standard variant. When a user switches to other variants, the annotations are no longer considered.

  • The filter values applied via the above logic are always cleared completely and overwritten by the incoming navigation context.

  • The filter fields that are applied as the default value are always made visible in the filter bar ‒ even if they are not configured as part of UI.SelectionFields.

Note

DisplayCurrency field also supports special handling. For more information, see section Handling default values from FLP in Navigation to an App (Inbound Navigation)
